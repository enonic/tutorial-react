= Link component
:sourcedir: ../
:imagesdir: media/

The `RichText` component imported from `@enonic/react-components` comes with a default component for rendering Links.

To see it in action let's add a link to `bio` field pointing to the other person's page:

1. Go to Content Studio and double click `Brad Pitt` inside Persons folder.
2. Select the `bio` field and add a link by pressing the *Link* button in the HTML area toolbar.
+
image::link-to-lea.png[width=888]
3. Set link text to `Link to Lea` and open the content dropdown.
4. Select `Lea Seydoux` from the list and press *Add*.
5. Press *Save* in the top toolbar and then *Mark as Ready* in the top toolbar to save and publish the changes.

The link should now be visible on the http://localhost:3000/p/brad-pitt/3a7b833f-3c37-4d55-88ae-cea21c556c68[Brad Pitt] page.

But when you click it you see: `URL doesn't exist!`

This is because our graphql data doesn't know about the URL structure of our webapp so we have to construct URLs following the pattern we declared in the <<routing#,routing chapter>>: `/p/:name/:personId`

This is one of the cases when custom `Link` component come in handy.

.src/main/resources/components/Person.tsx
[source, TypeScript]
----
import {Link} from './Link';

// ...

<RichText<RestProps>
  // ...
  Link={Link}
  // ...
/>
----

If you open http://localhost:3000/p/brad-pitt/3a7b833f-3c37-4d55-88ae-cea21c556c68[Brad Pitt's] page and click `Link to Lea` now you will be taken to the correct page!
