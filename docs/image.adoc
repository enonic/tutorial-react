= Image component
:sourcedir: ../

The `RichText` component imported from `@enonic/react-components` comes with a default component for rendering Images.

If you want to have more control over rendering RichText Images, you can provide your own customized `Image` component.

== Custom Image component

Here is an empty custom `Image` component example:

.src/components/Image.tsx
[source, TypeScript]
----
include::{sourcedir}src/main/resources/static/components/Image.tsx[]
----

Which you can import and pass to the `RichText` component like this:

.src/components/Person.tsx
[source, TypeScript]
----
import {Image} from './Image';

// ...

<RichText
  ...
  Image={Image}
  ...
/>
----

If you open http://localhost:3000/p/brad-pitt/3a7b833f-3c37-4d55-88ae-cea21c556c68[Brad Pitt] now, you will see that the images are still rendered, but now you can add custom logic.
